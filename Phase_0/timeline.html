Phase 04
AI Blast Radius Report
Wire Gemini into the simulation pipeline. After each simulation run, the structured output (affected services, faultline timeline, user impact %, recovery estimate) is passed to Gemini via tool calling — not a free-text prompt. Gemini returns a typed, structured blast radius report that is rendered in the frontend as a readable incident summary. This is how LLMs should be used in engineering systems: grounded in real data, not hallucinating.

🎤 Interview Answers You Now Own

"Tell me about a distributed systems project."
I built Faultline — a visual chaos engineering simulator. You draw your microservices graph, inject failures, and it simulates propagation using BFS traversal, real circuit breaker logic, and retry storm amplification. Gemini then generates a structured blast radius report. I learned more about failure modes building this than any course.
"How does a circuit breaker work?"
It's a 3-state FSM. CLOSED is normal — requests flow, failure rate tracked in a rolling window. When failures exceed threshold, it goes OPEN — all requests fail-fast immediately. After a timeout it goes HALF-OPEN, sends one probe. Success → CLOSED. Failure → OPEN again. I implemented this from scratch in Faultline's simulation engine.
"Why are retry storms dangerous?"
A slow service is more dangerous than a dead one. When slow, callers retry. 3 retries × 10 concurrent callers = 30x load. More load → more slowness → more retries. Exponential explosion. A dead service triggers circuit breakers and fails fast. I model this amplification factor in Faultline and visualize the spike on the RPS chart.
